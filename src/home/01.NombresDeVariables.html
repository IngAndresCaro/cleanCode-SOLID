Usar nombres que revelen las intenciones

El nombre de una variable, función o clase debe responder una serie de cuestiones básicas. Debe indicar por qué existe, qué hace y cómo se usa.

Ejemplo

Mal código

        public List<int[]> getThem() {
            List<int[]> list1 = new ArrayList<int[]>();
            for (int[] x : theList)
                if (x[0] == 4)
                    list1.add(x);
            return list1;
        }

Buen código

        public List<Celdas> obtenerCeldasMarcadas() {
            List<Celdas> celdasMarcadas = new ArrayList<Celdas>();
            for (Celdas celdas : juegoDeMesa)
                if (celdas.estaMarcada())
                    celdasMarcadas.add(celdas);
            return celdasMarcadas;
        }
